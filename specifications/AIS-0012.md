# AIS-0012 — Compliance

## Purpose

Compliance provides repeatable evidence that an implementation follows applicable AI-OPS requirements.

> **AIS-0012-REQ-001:** Every normative requirement MUST map to one or more test cases or documented manual assessments.

> **AIS-0012-REQ-002:** Test results MUST identify specification version, requirement ID, implementation version, environment and timestamp.

> **AIS-0012-REQ-003:** Result states MUST be `pass`, `fail`, `not_applicable`, `not_tested` or `inconclusive`.

> **AIS-0012-REQ-004:** A failure MUST include evidence and remediation guidance without exposing secrets.

> **AIS-0012-REQ-005:** Compliance levels MUST be declared by scope and MUST NOT imply coverage of untested specifications.

> **AIS-0012-REQ-006:** Automated tests MUST avoid destructive actions by default and MUST use fixtures, sandboxes or disposable environments.

> **AIS-0012-REQ-007:** Manual approval and safety-critical requirements MUST include verifiable audit evidence.

> **AIS-0012-REQ-008:** Compliance artifacts MUST be machine-readable and suitable for dashboard ingestion.

## Initial profiles

- `core`: AIS-0000 through AIS-0008;
- `mcp-aware`: core plus AIS-0009;
- `multi-agent`: mcp-aware plus AIS-0010;
- `platform`: multi-agent plus AIS-0011;
- `full-draft`: all current specifications.

## Scoring

A profile passes only when every applicable MUST requirement passes. SHOULD requirements are reported separately and do not compensate for failed MUST requirements.

## Evidence retention

Implementations MUST define retention, integrity and access policy for compliance reports.
