# AIS-0010 — AI Capability Registry

## Purpose

The registry describes what each AI agent can actually do in the current environment.

> **AIS-0010-REQ-001:** Each installed agent MUST have a registry record containing product, executable, version, installation method and update policy.

> **AIS-0010-REQ-002:** The record MUST identify available models and model gateways without exposing credentials.

> **AIS-0010-REQ-003:** MCP support, configured servers, native tools, shell access, filesystem scope, sandbox, network access and Git permissions MUST be recorded.

> **AIS-0010-REQ-004:** Approval behavior, unattended mode, write permissions and destructive capability MUST be classified.

> **AIS-0010-REQ-005:** Capability claims SHOULD be verified by safe probes rather than inferred only from product documentation.

> **AIS-0010-REQ-006:** The registry MUST represent Kimi Code, Claude Code, Codex CLI, OpenCode, Gemini CLI, Aider and unknown agents through a vendor-neutral schema.

> **AIS-0010-REQ-007:** Registry entries MUST include observation time, source and confidence.

> **AIS-0010-REQ-008:** An orchestrator MUST use registry data when assigning tasks and MUST NOT assign operations outside an agent's verified capability or permission boundary.

## Minimum record

`id`, `product`, `version`, `models`, `mcp`, `tools`, `permissions`, `sandbox`, `filesystem`, `network`, `git`, `approval`, `verified_at`, `confidence`.
