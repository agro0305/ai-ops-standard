# AIS-0009 — MCP Discovery

## Purpose

This document defines discovery and classification of Model Context Protocol servers and their operational risk.

> **AIS-0009-REQ-001:** Discovery MUST locate MCP configuration for every installed or in-scope agent.

> **AIS-0009-REQ-002:** Each server record MUST include name, transport, command or endpoint, package/image, version, working directory, environment variable names and enabled state.

> **AIS-0009-REQ-003:** Secret values MUST be redacted.

> **AIS-0009-REQ-004:** Tools, resources and prompts exposed by each server MUST be inventoried when protocol discovery is available.

> **AIS-0009-REQ-005:** Capabilities MUST be classified as read-only, state-changing, destructive, privileged, external-network or safety-critical.

> **AIS-0009-REQ-006:** Filesystem roots, repository scopes, database permissions and network reachability MUST be recorded.

> **AIS-0009-REQ-007:** Duplicate servers and overlapping capabilities SHOULD be detected.

> **AIS-0009-REQ-008:** Existing MCP entries MUST be preserved when adding or updating another server.

> **AIS-0009-REQ-009:** An agent MUST NOT connect all discovered MCP servers by default; least-capability profiles MUST be used.

> **AIS-0009-REQ-010:** MCP health and startup behavior MUST be tested without executing destructive tools.

## Registry

The inventory MUST support RepoWise, GitHub MCP, OpenAPI MCP, filesystem, databases, monitoring, Modbus, OPC UA and unknown custom servers without vendor-specific assumptions.
