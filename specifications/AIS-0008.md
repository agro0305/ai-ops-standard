# AIS-0008 — Rollback

## Purpose

Rollback restores an acceptable known state after failed, rejected or unsafe change.

> **AIS-0008-REQ-001:** A rollback procedure MUST be defined before execution.

> **AIS-0008-REQ-002:** Rollback triggers MUST include failed critical verification, unacceptable regression, loss of access, security degradation and explicit human request.

> **AIS-0008-REQ-003:** Rollback MUST use verified backups or documented inverse operations.

> **AIS-0008-REQ-004:** Rollback execution MUST be recorded with the same audit quality as forward execution.

> **AIS-0008-REQ-005:** The restored state MUST be verified after rollback.

> **AIS-0008-REQ-006:** Data-destructive rollback MUST require explicit approval unless required to prevent greater immediate harm.

> **AIS-0008-REQ-007:** If rollback fails, the agent MUST stop further non-recovery changes, preserve evidence and escalate.

> **AIS-0008-REQ-008:** The operation report MUST distinguish `not_required`, `available`, `started`, `completed`, `failed` and `manual_intervention_required`.

Rollback MUST NOT erase logs or evidence needed to diagnose the original failure.
