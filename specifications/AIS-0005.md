# AIS-0005 — Backup

## Purpose

Backup preserves recoverable pre-change state.

> **AIS-0005-REQ-001:** Existing files and configuration MUST be backed up before modification.

> **AIS-0005-REQ-002:** Backups MUST preserve permissions, ownership, timestamps and links when relevant.

> **AIS-0005-REQ-003:** Backup names MUST include a unique operation identifier or timestamp.

> **AIS-0005-REQ-004:** The agent MUST verify that the backup exists and is readable before execution.

> **AIS-0005-REQ-005:** Database, storage and distributed-system backups MUST use application-consistent methods when required for recovery.

> **AIS-0005-REQ-006:** Secrets in backups MUST receive protection at least equivalent to the source.

> **AIS-0005-REQ-007:** A backup MUST NOT be silently overwritten.

> **AIS-0005-REQ-008:** The report MUST record source, backup location, method, checksum when practical and restoration command.

A backup is not considered valid if no restoration path is known. When a recoverable backup cannot be created, the agent MUST classify the operation as high-risk and require explicit approval or stop.
