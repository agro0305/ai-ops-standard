# AIS-0003 — Discovery

## Purpose

Discovery establishes verified current state before planning or execution.

## Mandatory domains

> **AIS-0003-REQ-001:** Discovery MUST identify host, OS, kernel, architecture, user, privileges and time.

> **AIS-0003-REQ-002:** Discovery MUST inspect CPU, memory, disks, filesystems, mounts, storage health and free capacity.

> **AIS-0003-REQ-003:** Discovery MUST inspect interfaces, addresses, routes, DNS, listening ports, firewall, VPN and Tailscale state.

> **AIS-0003-REQ-004:** Discovery MUST inventory packages, runtimes, development tools and their versions.

> **AIS-0003-REQ-005:** Discovery MUST inventory systemd services, timers, cron jobs, containers, Compose projects, Podman and Kubernetes when present.

> **AIS-0003-REQ-006:** Discovery MUST identify reverse proxies, certificates, databases, message brokers, monitoring, logging and backup systems.

> **AIS-0003-REQ-007:** Discovery MUST locate Git repositories, dirty worktrees, remotes, branches and unpushed commits relevant to the operation.

> **AIS-0003-REQ-008:** Discovery MUST identify AI agents, model gateways, MCP servers, agent configuration files, permissions and capability boundaries.

> **AIS-0003-REQ-009:** Discovery MUST identify OpenAPI, TypeSpec, generated SDKs and API services relevant to the operation.

> **AIS-0003-REQ-010:** Secret values MUST be redacted while secret names and locations MAY be recorded.

## Port selection

An implementation MUST test actual listeners before selecting a port. It MUST NOT rely only on documentation or assumed defaults.

## Output

Discovery MUST produce a timestamped machine-readable inventory, evidence references, warnings, unknowns and confidence indicators.

## Failure handling

If required state cannot be established, the agent MUST stop or reduce the operation to read-only analysis.

## Compliance

Compliance requires evidence for all applicable discovery domains and explicit `not_present`, `not_applicable` or `unknown` values instead of silent omission.
