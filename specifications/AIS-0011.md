# AIS-0011 — Dashboard

## Purpose

The dashboard presents current inventory, operation status, risk and compliance without becoming the source of truth.

> **AIS-0011-REQ-001:** The dashboard MUST consume machine-readable AI-OPS records and MUST NOT invent missing state.

> **AIS-0011-REQ-002:** It MUST show host identity, observation time, inventory freshness, agent registry, MCP registry, service health, open operations and compliance status.

> **AIS-0011-REQ-003:** It MUST distinguish observed, inferred, stale, unknown and not-applicable values.

> **AIS-0011-REQ-004:** Operation views MUST show discovery, plan, approval, backups, changes, verification and rollback status.

> **AIS-0011-REQ-005:** Secrets and sensitive payloads MUST be redacted.

> **AIS-0011-REQ-006:** Read-only mode MUST be the default. Control actions MUST use authenticated, authorized backend APIs and explicit confirmation.

> **AIS-0011-REQ-007:** LAN and Tailscale deployment MUST be supported without requiring public internet exposure.

> **AIS-0011-REQ-008:** The dashboard MUST provide health and readiness endpoints and SHOULD expose OpenTelemetry or Prometheus-compatible metrics.

> **AIS-0011-REQ-009:** The UI SHOULD fit operational screens without unnecessary scrolling and SHOULD remain usable during partial backend failure.

## Core views

Overview, Inventory, Agents, MCP Servers, Operations, Backups, Verification, Compliance, Alerts and Audit.
