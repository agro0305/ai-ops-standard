# AIS-0007 — Verification

## Purpose

Verification determines whether the approved objective was achieved without unacceptable regression.

> **AIS-0007-REQ-001:** Verification MUST test resulting state, not only command exit status.

> **AIS-0007-REQ-002:** Success criteria defined in the plan MUST be checked individually.

> **AIS-0007-REQ-003:** Verification MUST include service health, configuration validity, logs and relevant functional tests.

> **AIS-0007-REQ-004:** Existing dependent services and network paths MUST be checked for regression when affected.

> **AIS-0007-REQ-005:** Tests MUST distinguish `passed`, `failed`, `skipped`, `not_applicable` and `unknown`.

> **AIS-0007-REQ-006:** A failed critical verification MUST trigger rollback evaluation and MUST prevent a success claim.

> **AIS-0007-REQ-007:** Verification evidence MUST be recorded with timestamps and redacted output.

> **AIS-0007-REQ-008:** Where practical, verification SHOULD be performed through an independent interface or method from the one used for execution.

## Result

Overall status is `passed`, `failed`, `partial` or `inconclusive`. Only `passed` permits an unconditional success statement.
