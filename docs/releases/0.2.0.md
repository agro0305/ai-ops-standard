# AI-OPS Standard 0.2.0

Release date: **2026-07-19**

Version 0.2.0 is the first validated reference implementation release candidate.

## Highlights

- read-only Linux, development-environment and AI capability discovery;
- deterministic compliance evaluation;
- authenticated local-first dashboard;
- report freshness, alerts, audit browsing and Prometheus metrics;
- atomic systemd-scheduled report refresh;
- incident lifecycle with active, acknowledged, silenced and resolved states;
- notification deduplication, cooldown and webhook support;
- backup and audit retention policies;
- operation plans with risk classification and explicit approval;
- verified backup manifests bound to the exact plan digest;
- dry-run-by-default execution and rollback;
- protected-root and symlink safeguards;
- non-destructive acceptance and runtime-health tooling.

## Validation evidence

The release candidate was validated on 2026-07-19 with:

- repository validation passed;
- **81 compliance tests passed**;
- release acceptance: **18/18 steps passed**;
- runtime health: **15/15 checks passed**;
- runtime warnings: **0**;
- runtime failures: **0**.

## Compatibility

The reference host used Python 3.12 and systemd. The implementation targets Linux and keeps state-changing behavior dry-run unless apply flags and required approvals are supplied.

## Important limitations

- Normative specifications remain Draft.
- Generic command actions require explicit rollback commands and declared backup targets.
- Package-manager, firewall, storage-controller and industrial-equipment adapters remain domain-specific.
- The dashboard is read-only and is not a control plane.
- Version 0.2.0 does not yet include the planned 0.3.0 policy engine and multi-host orchestration.

## Upgrade

Follow [Installation and Upgrade](../INSTALLATION.md). Run repository validation, the complete test suite, acceptance and runtime-health checks before considering an installation healthy.
